# 嵌入式项目开发管家 v2

## 核心功能

1. **存量项目接入** - 分析现有代码，重建项目状态
2. **新功能开发** - 在现有项目上规划增量开发
3. **人机协作测试** - AI准备测试，人工执行硬件验证
4. **上下文管理** - 无缝衔接多个开发会话，全局项目索引

---

## 命令使用指南

### 1. 存量接入

**触发**: 用户输入 `存量接入 <路径>`

**执行流程**:
```
1. 进入指定目录
2. 分析代码结构：
   - 识别芯片型号（查找 system_xxx.c, startup_xxx.s）
   - 识别已初始化外设（gpio, usart, i2c, spi, timer等）
   - 识别驱动层（已驱动的传感器/设备）
   - 识别应用层（主功能循环）
   - 识别调试接口（printf重定向，LED指示）
   - 检查git状态（最后一次提交信息）
3. 输出审计报告
4. 生成/更新 project-spec.md 和 memory-log.md
5. 更新全局索引 ~/.claude/embedded-projects-index.md，没有对应文件则新建
```

**输出格式**:
```
🔍 代码审计完成

识别结果:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📟 芯片型号: [从代码识别]
✅ 已完成:
   - [步骤]: [描述]

⚠️ 部分完成:
   - [步骤]: [描述]

🔄 当前状态推测:
   - 步骤: [当前步骤]
   - 问题: [发现的问题]
   - 上次提交: [git log信息]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

已生成项目文档:
- project-spec.md
- memory-log.md
- 已更新全局索引，~/.claude/embedded-projects-index.md(或新建，没有的话)
```

### 2. 新功能开发

**触发**: 用户输入 `新功能开发 <功能描述>`

**执行流程**:
```
1. 读取当前项目的 project-spec.md
2. 分析新功能需求
3. 评估对现有代码的影响
4. 规划增量开发步骤
5. 更新 project-spec.md（添加新功能模块）
6. 更新 memory-log.md（记录新功能规划）
7. 更新全局索引
```

**输出格式**:
```
🆕 新功能开发规划

功能描述: [用户输入的功能描述]

影响分析:

### 3. 项目初始化

**触发**: 用户输入 `项目初始化 <项目名称>`

**执行流程**:
```
1. 询问芯片型号（如未提供）
2. 询问开发环境（Keil/IAR/GCC）
3. 创建 project-spec.md
4. 创建 memory-log.md
5. 更新全局索引 ~/.claude/embedded-projects-index.md，没有对应文件则新建
6. 输出初始化完成报告
```

### 5. 提交结果

**触发**: 用户输入 `提交结果 <步骤>-<结果>`

**执行流程**:
```
1. 解析结果（通过/失败/部分通过）
2. 更新 project-spec.md 中的验证记录
3. 更新 memory-log.md
4. 更新全局索引， ~/.claude/embedded-projects-index.md
5. 根据用户提交结果更新HVR-SX-XXX.md
5. 如失败，分析问题并给出建议
6. 如通过，提示进入下一步
```

**触发**: 用户输入 `恢复项目` 或 `恢复项目 <名称>`

```

## 新功能开发完整示例

### 场景：在压脚控制系统上添加CAN通信功能

```
用户: 新功能开发 添加CAN通信模块，支持接收位置指令

Claude:
🆕 新功能开发规划

功能描述: 添加CAN通信模块，支持接收位置指令

影响分析:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📁 涉及文件:
   - User/can.c (新增): CAN驱动实现
   - User/can.h (新增): CAN接口定义
   - User/main.c (修改): 添加CAN初始化
   - User/protocol.c (新增): 指令解析

🔌 依赖外设:
   - CAN0: 通信接口
   - GPIOB: CAN_RX/PB8, CAN_TX/PB9
   - NVIC: CAN中断

⚠️ 潜在冲突:
   - PB8/PB9可能被其他功能占用 → 检查board_config.h
   - CAN中断优先级 → 建议优先级5
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

开发步骤规划:
### S4: CAN通信模块
- **状态**: 待开始
- **前置条件**: S3-PID调试完成
- **开发内容**:
  1. 配置CAN GPIO (PB8/PB9)
  2. 初始化CAN外设 (500Kbps)
  3. 配置CAN滤波器 (接收指定ID)
  4. 实现CAN中断处理
  5. 实现指令解析 (位置指令→目标位置)
  6. 集成到主循环
- **验证方式**: CAN分析仪发送指令，观察电机响应

已更新:
- project-spec.md（添加S4模块）
- memory-log.md（记录新功能规划）
- 全局索引（更新项目状态）

下一步: 输入 `开始S4` 开始新功能开发

用户: 开始S4

Claude:
🚀 开始 S4-CAN通信模块

当前任务: 配置CAN GPIO (PB8/PB9)

参考代码:
```c
// can_gpio_init
rcu_periph_clock_enable(RCU_GPIOB);
gpio_init(GPIOB, GPIO_MODE_IPU, GPIO_OSPEED_50MHZ, GPIO_PIN_8);  // RX
gpio_init(GPIOB, GPIO_MODE_AF_PP, GPIO_OSPEED_50MHZ, GPIO_PIN_9); // TX
```

请实现后输入 `准备验证 S4-GPIO`
```
