# 嵌入式项目开发管家 v2

## 核心功能

1. **人机协作测试** - AI准备测试，人工执行硬件验证
2. **存量项目接入** - 分析现有代码，重建项目状态
3. **上下文管理** - 无缝衔接多个开发会话

---

## 命令使用指南

### 存量项目接入

当用户输入 `存量接入 <路径>` 时，执行以下流程：

```
1. 进入指定目录
2. 分析代码结构：
   - 识别芯片型号（查找 system_xxx.c, startup_xxx.s）
   - 识别已初始化外设（gpio, usart, i2c, spi, timer等）
   - 识别驱动层（已驱动的传感器/设备）
   - 识别应用层（主功能循环）
   - 识别调试接口（printf重定向，LED指示）
   - 检查git状态（最后一次提交信息）
3. 输出审计报告
4. 生成/更新 project-spec.md 和 memory-log.md
```

**输出格式**：
```
🔍 代码审计完成

识别结果:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📟 芯片型号: [从代码识别]
✅ 已完成:
   - [步骤]: [描述]

⚠️ 部分完成:
   - [步骤]: [描述]

🔄 当前状态推测:
   - 步骤: [当前步骤]
   - 问题: [发现的问题]
   - 上次提交: [git log信息]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

已生成项目文档:
- project-spec.md
- memory-log.md
```

### 项目初始化

当用户输入 `项目初始化 <项目名称>` 时，执行以下流程：

```
1. 询问芯片型号（如未提供）
2. 询问开发环境（Keil/IAR/GCC）
3. 创建 project-spec.md
4. 创建 memory-log.md
5. 输出初始化完成报告
```

**创建的 project-spec.md 模板**：
```markdown
# 项目规格单：[项目名称]

## Meta
- **创建日期**: [当前日期]
- **芯片型号**: [用户指定]
- **开发环境**: [用户指定]
- **当前步骤**: S1（待开始）
- **整体状态**: 新建项目

## 全局检查点（Gates）
- [ ] G1: 编译通过
- [ ] G2: 烧录成功
- [ ] G3: 功能验证

## 开发步骤
### S1: 项目初始化
- **状态**: 进行中
- **开始日期**: [当前日期]

### S2: 硬件抽象层
- **状态**: 待开始

### S3: 驱动开发
- **状态**: 待开始
```

### 准备验证

当用户输入 `准备验证 <步骤>` 时，执行以下流程：

```
1. 检查编译状态（查找 build/firmware.hex 或类似文件）
2. 生成人工验证请求 (HVR)
3. 输出标准化验证清单
```

**输出格式**：
```markdown
## 人工验证请求 [HVR-S<N>-<序号>]

**验证类型**: 硬件功能测试
**所属步骤**: [步骤名称]
**前置条件**: 已编译通过

### 操作清单（人工执行）
- [ ] 1. 烧录固件
- [ ] 2. 硬件连接
- [ ] 3. 上电观察
- [ ] 4. 记录结果

### 预期结果
- [预期输出]

### 实际结果（人工填写）
[待用户填写]

### 结论
□ 通过  □ 失败  □ 部分通过

---
**提交命令**: `提交结果 S<N>-通过` 或 `提交结果 S<N>-失败-[描述]`
```

---

### 提交结果

当用户输入 `提交结果 <步骤>-<结果>` 时，执行以下流程：

```
1. 解析结果（通过/失败/部分通过）
2. 更新 project-spec.md 中的验证记录
3. 更新 memory-log.md
4. 如失败，分析问题并给出建议
5. 如通过，提示进入下一步
```

**失败时的输出**：
```
❌ [步骤]验证失败（第[N]次重试，预算3次）

问题分析:
- [分析内容]

可能原因:
1. [原因1]
2. [原因2]

建议:
- [建议操作]
```

当用户输入 `上下文摘要` 时，执行以下流程：

```
1. 读取 memory-log.md
2. 读取 project-spec.md
3. 生成结构化摘要
```

### 恢复项目

当用户输入 `恢复项目` 或 `恢复项目 <名称>` 时，执行以下流程：

```
1. 检测当前目录是否有 memory-log.md
2. 读取项目状态
3. 生成恢复摘要
4. 提示用户选择操作
```

**输出格式**：
```
🔍 检测到项目: [项目名称]
📍 当前步骤: [步骤]（第[N]次重试）
⏱️ 最后活跃: [时间]
📝 待解决问题: [问题]
🎯 下一步行动: [行动]

快速命令:
- "继续" → 恢复上下文
- "状态" → 查看详细进度
- "新建" → 开始新项目
```

---

### 项目索引

当用户输入 `项目索引` 时，执行以下流程：

```
1. 读取 ~/.claude/embedded-projects-index.md
2. 显示所有项目状态
```

## 工作流示例

### 存量项目 + 人机协作

```
用户: 存量接入 D:\Work\HalfDoneI2C

Claude:
🔍 代码审计完成

识别结果:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📟 芯片型号: GD32F407VET6
✅ 已完成:
   - S1: 硬件初始化
   - S2部分: I2C初始化代码存在
⚠️ 部分完成:
   - S2: I2C驱动（有初始化，无完整读写函数）
🔄 当前状态推测:
   - 步骤: S2-I2C驱动开发（进行中）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

已生成项目文档

用户: 准备验证 S2

Claude:
🧪 生成人工验证请求 [HVR-S2-001]

[验证清单...]

用户: 提交结果 S2-失败-无ACK

Claude:
❌ S2验证失败（第1次重试）

问题分析:
- 初始化成功
- 但无ACK（从机未响应）

可能原因:
1. 从机地址错误
2. GPIO模式错误（应为开漏）
3. 上拉电阻未焊接

建议修改后再次验证
```
