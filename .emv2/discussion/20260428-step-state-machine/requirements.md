# 需求确认 - 步骤状态自动推进

## 已确认参数

### 1. 当前步骤显示格式
- **格式**: `S<编号>-<子步骤>(<状态标签>)`
- 示例: `S7-B(开发中)`
- 状态标签位置：括号内

### 2. 推进时机
- 任何完成操作都触发推进（讨论确认、开发完成、验证通过等）
- 包括 `/em result` 和其他状态变更操作

### 3. 状态标签体系

| 标签 | 含义 | 使用场景 |
|------|------|---------|
| 🔲 待开发 | 已规划未开始 | 新创建的子步骤 |
| 🚧 开发中 | 正在开发 | 开始开发 → 验证前 |
| 🔄 验证中 | 正在验证 | verify 命令发出后 |
| ✅ 完成 | 已完成 | 验证通过 |
| 🔁 返工中 | 失败重做 | 验证失败进入返工 |

### 4. 主步骤完成条件
- **B**: 人工确认 — 所有子步骤完成后，手动执行 `/em result s7-通过` 标记主步骤完成
- 不自动判定
