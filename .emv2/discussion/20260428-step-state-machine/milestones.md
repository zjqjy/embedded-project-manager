# 子流程拆分 - 步骤状态自动推进

## 执行步骤

| 步骤 | 子功能 | 优先级 | 说明 | 验证方式 |
|------|--------|--------|------|---------|
| **A** | `project-spec.md` 状态格式 | P0 | 定义 `当前步骤: S7-A(开发中)` 格式，子步骤状态标签表 | 人工核对格式 |
| **B** | `commands/result.md` 推进逻辑 | P0 | 通过 → ✅ 完成 + 推进下一子步骤；失败 → 🔁 返工中 | 模拟提交结果观察状态变化 |
| **C** | `commands/verify.md` 状态变更 | P0 | verify 发出 → 🚧 开发中 → 🔄 验证中 | 模拟 verify 观察状态变化 |
| **D** | `workflows/hvr-workflow.md` 流程图 | P1 | 在流程图各节点加入状态变更说明 | 人工核对流程 |

## 状态机规则

### 状态流转图
```
🔲 待开发 ──(/em new 完成后)──→ 🚧 开发中
🚧 开发中 ──(/em verify)──────→ 🔄 验证中
🔄 验证中 ──(/em result 通过)──→ ✅ 完成
🔄 验证中 ──(/em result 失败)──→ 🔁 返工中
🔁 返工中 ──(/em verify)──────→ 🔄 验证中
✅ 完成 ──(自动)─────────────→ 下一子步骤 🔲 待开发
```

### Meta 区格式
```
当前步骤: S7-A(开发中)
```
