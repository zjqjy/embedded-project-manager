# S5 串口调试工具 - 需求确认（最终版）

## 核心定位

**S5 = MCP工具 + GUI + 人-AI协作验证流程**

```
┌─────────────────────────────────────────────────────────────┐
│  Serial Monitor & AI Assistant (S5核心产品)                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐   │
│  │  串口配置区  │  │  日志显示区  │  │   AI分析区      │   │
│  │  端口/波特率 │  │  实时显示   │  │   分析结果      │   │
│  └─────────────┘  └─────────────┘  └─────────────────┘   │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  发送区: [输入命令...] [发送]                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  MCP接口 ──→ Claude Code读取数据 ──→ AI分析 ──→ 协作诊断  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 人-AI协作验证模型

```
人类执行验证 ←→ 设备 ←→ 串口 ←─→ S5工具(UI+AI通道)
       ↓                    ↓
   观察现象      →    AI接收日志    →
       ↓                    ↓
   人类口述 ─────→ AI记录分析 ←────
       ↓
   共同决策
```

## 功能清单

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 串口配置 | 端口、波特率可配置 | P0 |
| 实时显示 | 串口数据实时显示 | P0 |
| 发送命令 | 向串口发送命令 | P0 |
| 日志保存 | 保存到文件供追溯 | P0 |
| MCP接口 | 供Claude读取数据 | P0 |
| 时间戳 | 每行数据加时间戳 | P1 |
| AI分析区 | 显示AI分析结果 | P1 |

## 技术方案

| 项目 | 选择 |
|------|------|
| GUI框架 | tkinter（Python内置） |
| 串口库 | pyserial |
| MCP SDK | Python版MCP SDK |
| 日志格式 | `.emv2/logs/serial_<步骤>_<时间>.log` |

## 子流程拆分

### S5-A: MCP工具开发
- MCP Server + tkinter UI
- 串口配置（端口、波特率）
- 收发数据 + 实时显示
- 日志文件保存

### S5-B: EM流程集成
- `/em verify sX` → 打开工具
- MCP接口供AI读取日志
- HVR增强版对接

### S5-C: 实际验证测试
- 用增强版HVR进行实际验证

---

## 交互原则

- **人类只做**：执行验证操作、口头描述观察
- **AI负责**：记录到HVR文件、分析根因、给出建议、维护文档
- **HVR是AI维护、人类确认的工作区**，不是人类自己编辑的文件
