# S5 子流程拆分（最终版）

## S5: Serial Monitor & AI Assistant

### S5-A: MCP工具开发
- **所属子系统**: MCP工具+UI
- **开发内容**:
  1. tkinter界面设计（配置区/显示区/发送区/AI区）
  2. pyserial串口通信
  3. MCP Server接口
  4. 日志文件保存
- **前置条件**: 无
- **验证方式**: 工具可运行，串口收发正常
- **优先级**: 1

### S5-B: EM流程集成
- **所属子系统**: EM工作流
- **开发内容**:
  1. `/em verify sX` → 打开S5工具
  2. MCP接口供AI读取日志
  3. HVR增强版格式
  4. 协作诊断流程
- **前置条件**: S5-A完成
- **验证方式**: 实际项目验证
- **优先级**: 2

### S5-C: 实际验证测试
- **所属子系统**: 验证
- **开发内容**:
  1. 用增强版HVR进行实际验证
  2. 测试人类口述→AI记录流程
  3. 测试共同决策流程
- **前置条件**: S5-B完成
- **验证方式**: 实际项目验证
- **优先级**: 3

## 完整验证提交流程

```
┌─────────────────────────────────────────────────────────────┐
│                    阶段1: 准备验证                           │
├─────────────────────────────────────────────────────────────┤
│  /em verify s3 → 打开S5工具                                │
│       ↓                                                    │
│  AI生成 HVR-S3-001.md（增强版）                            │
│       ↓                                                    │
│  输出验证清单                                               │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│                 阶段2: 执行验证+观察                         │
├─────────────────────────────────────────────────────────────┤
│  人类: 在S5工具配置串口 → 连接 → 执行验证                   │
│       ↓                                                    │
│  S5工具: 实时显示日志                                       │
│       ↓                                                    │
│  人类: 观察物理世界 → 口述现象                             │
│       ↓                                                    │
│  AI: 调用MCP读取日志 → 记录观察 → 分析根因                 │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│               阶段3: 共同决策（关键）                         │
├─────────────────────────────────────────────────────────────┤
│         ┌───────────┬───────────┬───────────┐              │
│         ↓           ↓           ↓           ↓              │
│      【通过】    【失败】    【部分】   【不确定】            │
│         │           │           │           │              │
│         ↓           ↓           ↓           ↓              │
│    记录结果    分析根因    记录现状   进一步分析              │
│    准备提交    制定方案    继续调试   人类补充信息            │
└─────────────────────────────────────────────────────────────┘
```

## 四种决策分支

| 决策 | 人类输入 | AI行为 |
|------|----------|--------|
| 通过 | "通过了" | 记录→更新project-spec→提示下一步 |
| 失败 | "失败了，现象是XXX" | 记录→分析→制定方案→人类修改→重验 |
| 部分 | "部分功能正常" | 记录部分成功→识别问题→决定后续 |
| 不确定 | "不知道正不正常" | 询问细节→人类补充→消除不确定 |
