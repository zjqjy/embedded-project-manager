# 子流程拆分 - 优化项目记忆和工具文件模板

## 执行步骤

| 步骤 | 子功能 | 优先级 | 说明 |
|------|--------|--------|------|
| **A** | `project-spec.md` 瘦身 | P0 | 去掉决策/讨论摘要/待办，只留步骤表+技术方案，讨论区只留引用 |
| **B** | `memory-log.md` 统一 | P0 | 去掉步骤状态表和「最近更新」，统一用会话历史 |
| **C** | `SKILL.md` 精简 | P1 | 去掉命令列表，保留路由逻辑 |
| **D** | `commands/*.md` 瘦身 | P1 | 去掉细则/模板/流程图，引用 workflows/，保留触发+流程概要 |
| **E** | `workflows/*.md` 结构对齐 | P1 | 检查不重复、不矛盾 |
| **F** | `help.md` 独立维护命令列表 | P1 | 命令列表唯一来源改为 help.md |

## 核心原则
一条信息只在一个地方维护
