# 需求讨论: 20260416-auto-read-emv2

## 需求描述

对于管理助手生成的 `.emv2` 下的文件（除 `discussion/` 外），读取时不需要人类自动确认。

## 需求拆分

### 子系统

#### 1. 信任机制模块
- **类型**: 配置/工具
- **简述**: 检查 `~/.claude/settings.json` 中的 permissions 配置

#### 2. 路径解析模块
- **类型**: 配置/工具
- **简述**: 将项目路径下的 `.emv2/**/*` 转换为绝对路径匹配规则

---

## 需求确认

| 项目 | 值 |
|------|-----|
| **目标** | 读取 `.emv2` 下文件时不需要人工确认 |
| **检查位置** | `~/.claude/settings.json` 的 `permissions.allow` |
| **配置格式** | permissions.allow 数组 |
| **生效范围** | `.emv2` 下除 `discussion/` 外的所有文件 |

---

## 硬件对齐

此需求为配置文件类，不涉及硬件。

## 头脑风暴

### 难点1: settings.json 可能不存在
- **方案**: 读取现有文件，追加配置，备份原文件

### 难点2: 如何检查现有配置
- **方案**: 解析 JSON 检查是否已有 .emv2 相关规则

### 难点3: 触发时机
- **方案**: 首次使用 EM-SKILL 时检查，全局只执行一次

## 配置规则（最终）

| 规则 | 匹配范围 |
|------|----------|
| `Read(./.emv2/*.md)` | 根目录 .md 文件 |
| `Read(./.emv2/*.json)` | 根目录 .json 文件 |
| `Read(./.emv2/checkpoints/**/*)` | checkpoints/ 所有文件 |

| `Read(./.emv2/history/**/*)` | history/ 所有文件 |

| **不配置** | discussion/ 保持需确认 |
