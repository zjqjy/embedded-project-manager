# 需求确认 - new 命令步骤编号

## 已确认参数

### 1. 编号格式
- **格式**: `S<数字>-<描述标签>`（如 S3-GUI、S7-CAN通信）
- 数字部分从 1 开始递增
- 标签部分使用功能简短描述（英文或拼音/中文均可）

### 2. 编号存储位置
- **存储位置**: `project-spec.md` 的「开发步骤状态」表中
- 读取时遍历当前表获取最大 S 编号
- 不新增独立计数器文件

### 3. `/em new` 修改逻辑
- 自动读取 project-spec.md → 解析步骤表 → 取最大 S 编号
- 新步骤编号 = 当前最大编号 + 1
- 格式对齐现有步骤表（步骤 | 名称 | 状态 | 日期）
- 写入 project-spec.md 步骤表，状态为「🚧 讨论中」

### 4. 存量补号
- 需要对当前 project-spec.md 中已有步骤追溯补号
- 当前 S1-S6 已有编号，只需核对一致性
- 新步骤从 S7 开始
