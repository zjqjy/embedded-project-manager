# 子流程拆分 - new 命令步骤编号

## 执行步骤

| 步骤 | 子功能 | 优先级 | 说明 | 验证方式 |
|------|--------|--------|------|---------|
| **A** | `new.md` 编号逻辑 | P0 | 新增自动读取 project-spec.md → 取最大 S 编号 → 分配 S(N+1) | 运行 `/em new` 观察编号是否正确 |
| **B** | `project-spec.md` 存量补号 | P0 | 检查当前步骤表编号，确保 S1-S6 连续完整 | 人工核对 |
| **C** | `stat.md` 兼容 | P1 | 确保 stat 能正确解析带 S 编号的步骤 | 运行 `/em stat` 观察 |
| **D** | `verify.md` 引用对齐 | P1 | verify 中引用的步骤 ID 与 new 生成的编号一致 | 运行 `/em verify s7` 观察 |

## 设计要点

- 编号格式: `S<数字>`（如 S7）
- 分配方式: `/em new` 自动读取 project-spec.md 最大编号 +1
- 废弃编号: 不复用
- 存储位置: project-spec.md 步骤表
- 存量起点: 当前项目 S6 已完成，下一个从 S7 开始
