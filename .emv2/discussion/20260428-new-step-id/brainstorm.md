# 头脑风暴 - new 命令步骤编号

## 确认方案

### 编号分配策略
- **方案**: `/em new` 内部自动分配编号（方案 A）
- **废弃编号**: 不复用，保持唯一
- **验证引用格式**: `S<数字>` 即可，不带描述标签

### 关键设计点

1. **自动分配逻辑**
   - `/em new` 读取 project-spec.md → 解析步骤表
   - 正则提取所有 `S<数字>` → 取最大值 → +1
   - 若步骤表为空（新项目），从 S1 开始

2. **写入格式**
   ```
   | S7 | <功能描述> | 🚧 讨论中 | <日期> |
   ```

3. **存量补号**
   - 当前 project-spec.md 中 S1-S6 已有编号，确认一致
   - 合入后新建步骤从 S7 开始
