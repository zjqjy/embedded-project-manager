# 需求拆分 - 20260602-skill-standardize-universal-git

## 原始需求
将本技能组织为标准模式用创建技能skill,然后需要把本技能改造成通用skill 同时融入git进工作流

## 拆分结果（4 个子系统）

### 1. Skill 标准化
- **类型**：其他（架构重组）
- **简述**：用"创建技能"skill 的标准模式重组织目录结构、SKILL.md、commands/、workflows/、templates/

### 2. Skill 通用化
- **类型**：其他（领域抽象）
- **简述**：抽象掉嵌入式专属假设，使 EM-SKILL 适用于任意工程类项目

### 3. Git 工作流集成
- **类型**：其他（流程增强）
- **简述**：在 HVR/验证/归档/步骤推进节点融入 git 操作（commit/branch/tag/CHANGELOG）

### 4. 嵌入式能力保留
- **类型**：其他（根能力保留）
- **简述**：EM-SKILL 诞生即是为嵌入式，串口/烧录/芯片学习/embed-ai-tool 整合等核心能力必须完整保留，作为"原生场景"

## 设计原则
- **1-3 改造**让 EM 走向通用
- **4 兜底**让嵌入式场景不退化、不丢功能

## 用户确认
- ✅ 用户确认 4 个子系统结构
- ✅ "嵌入式这个能力还是要保留了，因为一开始就是为它而生不能丢了" — 子系统 4 升级为独立子系统（非约束）

## 状态
- 阶段 1 (需求拆分) - ✅ 完成
- 阶段 2 (需求确认) - ⏳ 进行中
