# 命令: /em result (提交结果)

## 功能
提交验证结果，触发后续流程

## 触发
```
/em result <步骤>-<结果>
```

## 执行流程

### 通过时
```
1. 解析结果，提取步骤编号（如 s7-a）
2. 🟢 更新当前子步骤状态：🔄 验证中 → ✅ 完成
3. 🟢 自动推进到下一子步骤：
   - 读取子步骤表，找到当前步骤的下一个
   - 更新 Meta 当前步骤为下一子步骤(开发中)
   - 如无下一子步骤 → 主步骤完成，提示 /em result s7-通过
4. 更新 project-spec.md 验证记录
5. 更新 memory-log.md
6. 如有闭环问题 → 更新 problem-log.md 闭环记录
7. 自动归档（如有必要）
8. 提示下一步开发或验证
9. 更新全局索引
```

### 失败时
```
1. 解析失败信息，提取步骤编号（如 s7-a）
2. 🔴 更新当前子步骤状态：🔄 验证中 → 🔁 返工中
3. 🔴 更新 Meta 当前步骤：S7-A(返工中)
4. 更新 HVR 文件记录失败
5. 创建 problem-log.md 条目
6. 进入问题讨论模式
7. 分析问题根因
8. 记录讨论结论和解决方案
```

## 关键提醒

⚠️ **失败时必须先更新 HVR 文件，再进入讨论模式**
