# 命令: /em result (提交结果)

## 功能
提交验证结果，触发后续流程

## 触发
```
/em result <步骤>-<结果>
```

## 执行流程

### 通过时
```
1. 解析结果
2. 更新 project-spec.md 验证记录
3. 更新 memory-log.md
4. 如有闭环问题 → 更新 problem-log.md 闭环记录
5. 自动归档（如有必要）
6. 提示进入下一步
7. 更新全局索引
```

### 失败时
```
1. 解析失败信息
2. 更新 HVR 文件记录失败
3. 创建 problem-log.md 条目
4. 进入问题讨论模式
5. 分析问题根因
6. 记录讨论结论和解决方案
```

## 失败触发流程 (HVR工作流)

```
用户提交: /em result s3-失败-描述
   ↓
1. 更新 HVR 文件（先执行！）
   ↓
2. 创建 problem-log.md 条目
   ↓
3. 进入问题讨论模式
   ↓
4. 分析问题：
   - 失败现象是什么？
   - 可能原因？
   - 排查方向？
   ↓
5. 记录解决方案
   ↓
6. 用户确认后实施修改
   ↓
7. 用户提交: /em result s3-通过
   ↓
8. 闭环记录 → resolved
```

## 关键提醒

⚠️ **失败时必须先更新 HVR 文件，再进入讨论模式**
