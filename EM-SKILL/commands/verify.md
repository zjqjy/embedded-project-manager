# 命令: /em verify (准备验证)

## 功能
生成人工验证请求 (HVR)

## 触发
```
/em verify <步骤>
```

## 执行流程

1. 读取项目当前步骤
2. 生成 HVR 文件
3. 保存到 `.emv2/checkpoints/HVR-<步骤>-<序号>.md`
4. 输出验证清单

## HVR 文件格式

```markdown
## 人工验证请求 [HVR-S<N>-<序号>]

**验证类型**: [硬件功能测试/Skill功能测试]
**所属步骤**: [步骤名称]
**前置条件**: [已编译通过/其他]
**创建时间**: {{DATETIME}}

### 操作清单（人工执行）
- [ ] 1. [操作1]
- [ ] 2. [操作2]

### 预期结果
- [预期1]
- [预期2]

### 实际结果（人工填写）
[待用户填写]

### 结论
□ 通过  □ 失败  □ 部分通过

---
**提交命令**: /em result S<N>-通过 或 /em result S<N>-失败-[描述]
```

## 工作流位置

```
用户: /em verify s3
   ↓
AI: 生成HVR文件
   ↓
输出验证清单
   ↓
用户: 人工执行验证
   ↓
用户: /em result s3-通过/失败
```
