# 命令: /em disc (讨论流程)

## 功能
进入需求对齐讨论流程

## 触发
```
/em disc [讨论ID]
```

## 讨论阶段

### 阶段1: 需求拆分
- 分析涉及的子系统/模块
- 用户确认/修改拆分结果
- 保存到 `discussion/<需求ID>/split.md`

### 阶段2: 需求确认
- 对每个子功能确认详细参数
- 按类型问相应问题（通信/测量/控制/状态机）
- 保存到 `discussion/<需求ID>/requirements.md`

### 阶段3: 硬件对齐
- 对涉及硬件的子功能确认引脚和配置
- 检查引脚冲突
- 保存到 `discussion/<需求ID>/hardware.md`

### 阶段4: 头脑风暴
- 讨论技术难点和解决方案
- 保存到 `discussion/<需求ID>/brainstorm.md`

### 阶段5: 子流程拆分
- 拆分为 S[X]-子功能 形式
- 确定验证方式
- 保存到 `discussion/<需求ID>/milestones.md`
- 更新 project-spec.md

## 讨论形式

- **明确选项**: AI列出选项，用户确认/选择
- **AI引导提问**: AI通过提问引导用户思考不明确的需求
- **确认即保存**: 用户确认后AI直接保存，不需要二次确认

## 状态转换

| 用户输入 | AI响应 |
|---------|--------|
| `继续` | 保存当前结果 → 进入下一阶段 |
| `跳过` | 跳过当前子功能 → 进入下一个 |
| `退出` | 保存进度 → 退出讨论 |
| `继续讨论` | 从上次位置继续 |

## 输出格式

```
📋 需求对齐讨论

讨论ID: <需求ID>
当前阶段: 阶段1/5 (需求拆分)

━━━━━━━━━━━━━━━━━━━━━━━━━━
[阶段内容]
━━━━━━━━━━━━━━━━━━━━━━━━━━

讨论进度: ██░░░░░░░░ 20%
阶段: 1/5 (需求拆分)

输入 `继续` 进入下一阶段
输入 `跳过` 跳过当前阶段
输入 `退出` 结束讨论
```
