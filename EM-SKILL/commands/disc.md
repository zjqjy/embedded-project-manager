# 命令: /em disc (讨论流程)

## 功能
进入需求对齐讨论流程（5阶段）

## 触发
```
/em disc [讨论ID]
```

## 讨论阶段概要

| 阶段 | 产出文件 | 说明 |
|------|---------|------|
| 1. 需求拆分 | `split.md` | 分析子系统/模块，用户确认 |
| 2. 需求确认 | `requirements.md` | 确认各子功能详细参数 |
| 3. 硬件对齐 | `hardware.md` | 确认引脚和配置，检查冲突 |
| 4. 头脑风暴 | `brainstorm.md` | 讨论技术难点和解决方案 |
| 5. 子流程拆分 | `milestones.md` | 拆分为S[X]子步骤，更新project-spec |

## 强制完成约束

**在讨论完成（阶段5）之前，禁止执行以下操作**：
- ❌ 编写任何代码、创建或修改文件
- ❌ 执行 `/em result` 或 `/em verify`
- ❌ 完成任务（完成开发步骤）

**如果用户要求提前执行**：
AI 应回复："讨论尚未完成（X/5阶段），请先完成讨论再执行后续操作。输入 `继续` 进入下一阶段。"

## 讨论形式

- **AI列出选项** → 用户确认/选择
- **AI引导提问** → 用户回答
- **确认即保存**：用户确认后AI直接保存
