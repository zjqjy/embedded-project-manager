# 命令: /em new (新功能开发)

## 功能
规划增量开发（必须先讨论）

## 触发
```
/em new <功能描述>
```

## 重要
新功能开发**必须先进行需求对齐讨论**，不能跳过。

## 执行流程

1. 读取 `.emv2/project-spec.md`
2. 分析新功能需求
3. 生成需求ID（格式: `<日期>-<简短描述>`）
4. **自动进入**需求对齐讨论流程（/em disc）
5. 讨论完成后，评估对现有代码的影响
6. 规划增量开发步骤
7. 更新全局索引

## 输出格式

```
🆕 新功能开发

功能描述: [用户输入的功能描述]
讨论ID: [需求ID]

━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 需求对齐讨论（自动进入）

讨论ID: <需求ID>
当前阶段: 1/5 (需求拆分)
...
```
