# 命令: /em new (新功能开发)

## 功能
规划增量开发（必须先讨论）

## 触发
```
/em new <功能描述>
```

## 重要
新功能开发**必须先进行需求对齐讨论**，不能跳过。

## 执行流程

1. 读取 `.emv2/project-spec.md`
2. 解析「开发步骤状态」表，提取所有 `S<数字>` 编号
3. 取最大 S 编号 → 新步骤编号 = 当前最大 + 1
   - 例如当前最大 S6，新步骤编号为 S7
   - 项目无存量步骤时从 S1 开始
4. 分析新功能需求
5. 生成需求ID（格式: `<日期>-<简短描述>`）
6. **自动进入**需求对齐讨论流程（/em disc）
7. 讨论完成后，评估对现有代码的影响
8. 规划增量开发步骤，分配上一步计算的 S 编号
9. 更新 `.emv2/project-spec.md`（添加新功能模块，写入步骤表）
10. 更新 `.emv2/memory-log.md`（记录新功能规划）
11. 更新全局索引

## 输出格式

```
🆕 新功能开发

功能描述: [用户输入的功能描述]
步骤编号: S<自动分配>
讨论ID: [需求ID]

━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 需求对齐讨论（自动进入）

讨论ID: <需求ID>
当前阶段: 1/5 (需求拆分)
...
```

## 步骤编号规则

- 编号格式: `S<数字>`（如 S7）
- 分配方式: 自动读取 project-spec.md 中最大编号 +1
- 废弃步骤编号不复用
- verify 时引用格式: `S<数字>`（如 `/em verify s7`）
