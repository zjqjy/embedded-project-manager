# 命令: /em new (新功能开发)

## 功能
规划增量开发（必须先讨论）

## 触发
```
/em new <功能描述>
```

## 重要
新功能开发**必须先进行需求对齐讨论**，不能跳过。

## 执行流程

1. **【状态目录检测】** 调用 `get_state_dir()` 确定 `<STATE_DIR>`（S10-B 通用化）
2. 读取 `<STATE_DIR>/project-spec.md`
3. 解析「开发步骤状态」表，提取所有 `S<数字>` 编号
4. 取最大 S 编号 → 新步骤编号 = 当前最大 + 1
   - 例如当前最大 S6，新步骤编号为 S7
   - 项目无存量步骤时从 S1 开始
5. 分析新功能需求
6. 生成需求ID（格式: `<日期>-<简短描述>`）
7. **自动进入**需求对齐讨论流程（/em disc）
8. 讨论完成后，评估对现有代码的影响
9. 规划增量开发步骤，分配上一步计算的 S 编号
10. 更新 `<STATE_DIR>/project-spec.md`（添加新功能模块，写入步骤表）
11. 更新 `<STATE_DIR>/memory-log.md`（记录新功能规划）
12. 更新全局索引

> 📌 **目录通用化（S10-B）**：所有 `.emv2/` 引用替换为 `<STATE_DIR>/`，
> 实际路径由 `get_state_dir()` 解析。新项目为 `.em/`，旧 `.emv2/` 项目零配置兼容。

## 输出格式

```
🆕 新功能开发

功能描述: [用户输入的功能描述]
步骤编号: S<自动分配>
讨论ID: [需求ID]

━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 需求对齐讨论（自动进入）

讨论ID: <需求ID>
当前阶段: 1/5 (需求拆分)
...
```

## 步骤编号规则

- 编号格式: `S<数字>`（如 S7）
- 分配方式: 自动读取 project-spec.md 中最大编号 +1
- 废弃步骤编号不复用
- verify 时引用格式: `S<数字>`（如 `/em verify s7`）
